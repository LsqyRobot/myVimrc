# 🚀 自动安装模式指南

## 什么是自动安装模式？

自动安装模式是一个**一键安装**选项，无需任何用户交互，直接安装极致现代型 Neovim 配置。

## 🎯 适用场景

### 推荐使用自动模式的情况
- ✅ **快速体验** - 想要快速试用配置
- ✅ **CI/CD 环境** - 自动化部署脚本
- ✅ **批量安装** - 多台机器配置
- ✅ **不需要个人信息** - 不想设置作者姓名等信息
- ✅ **默认选择** - 接受所有推荐设置

### 使用交互模式的情况
- ⚙️ **自定义配置** - 需要选择传统 Vim
- ⚙️ **个人信息** - 想要配置作者姓名、邮箱等
- ⚙️ **了解细节** - 想要了解安装过程

## 📋 使用方法

### 自动安装 ⭐
```bash
cd /home/lucas/myVimrc
./build.sh --auto
```

### 查看帮助
```bash
./build.sh --help
```

### 其他选项
```bash
./build.sh                    # 交互式安装
./build.sh --upgrade-neovim   # 升级 Neovim
```

## 🔍 自动模式具体做什么？

### 自动选择的配置
- **编辑器类型**: 极致现代型 Neovim
- **语言支持**: C/C++/Python
- **插件安装**: 所有推荐插件
- **个人信息**: 跳过配置

### 安装的组件
1. **系统依赖**: neovim, clang-format, python3-yapf, etc.
2. **插件管理器**: vim-plug
3. **核心插件**: LSP、补全、搜索、文件树、终端等
4. **语言服务器**: clangd (C/C++), pyright (Python)
5. **语法高亮**: Treesitter 解析器

### 跳过的步骤
- ❌ 编辑器类型选择
- ❌ 安装确认
- ❌ 个人信息配置

## ⚡ 安装时间对比

| 模式 | 预计时间 | 用户交互 |
|------|----------|----------|
| 自动模式 | 3-5 分钟 | 0 次 |
| 交互模式 | 5-8 分钟 | 3-5 次 |

## 🛠️ 安装后配置

自动安装完成后，你可能需要：

### 1. 测试功能
```bash
nvim
# 测试基本功能：
# - Space+e (文件树)
# - Space+ff (搜索文件)
# - gd (跳转定义)
```

### 2. 配置 LSP 服务器
```vim
:Mason
# 检查 clangd 和 pyright 是否已安装
```

### 3. 配置 Copilot (可选)
```vim
:Copilot setup
# 需要 GitHub 账号和订阅
```

### 4. 后续个人信息配置 (可选)
如果后来需要添加个人信息，可以手动编辑配置文件：
```bash
# 编辑配置文件
nvim ~/.config/nvim/init.vim

# 添加以下内容：
" ===== 个人信息配置 =====
let g:header_field_author = 'Your Name'
let g:header_field_author_email = 'your.email@example.com'
let g:header_field_copyright = 'Copyright (c) 2025'
```

## ❓ 常见问题

### Q: 自动模式能否后续修改配置？
A: 可以！所有配置文件都可以手动编辑，也可以重新运行交互模式。

### Q: 如何从自动模式切换到传统 Vim？
A: 重新运行 `./build.sh` 并选择选项 1。

### Q: 自动模式是否安装所有插件？
A: 是的，安装所有推荐的核心插件，但跳过可选的个人信息配置。

### Q: 出错了怎么办？
A: 查看错误信息，通常重新运行即可。如果持续出错，使用交互模式排查。
